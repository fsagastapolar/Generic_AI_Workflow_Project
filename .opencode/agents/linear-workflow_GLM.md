## State Resolution

State UUIDs are workspace-specific. Resolve them dynamically once per session and cache:

```bash
curl -s -X POST https://api.linear.app/graphql \
  -H "Authorization: $LINEAR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query": "query { team(id: \"'$LINEAR_TEAM_ID'\") { states { nodes { id name type } } } }"}' | jq '.data.team.states.nodes[] | "\(.name) → \(.id)"'
```

Map the config's `workflowStates` values to the resolved UUIDs. If a state name from config doesn't match any workspace state, report a warning.

## Actions

### implement_start

1. Resolve state UUID for the "In Progress" state (from config `workflowStates.inProgress`)
2. Move ticket to "In Progress"
3. Post comment: `🤖 **AI Implementation Log** — Implementation started on branch \`<branch>\`. Plan: \`<path>\``
4. Return: `SUCCESS` or `WARNING: <details>`

### implement_progress

- If normal phase completion, post comment: `🤖 **AI Implementation Log** — Phase N: [Name] completed. [Summary]`
- If difficulty, post comment: `🤖 **AI Implementation Log** — ⚠️ [Description]. [Resolution]`
- No state transition
- Return: `SUCCESS` or `WARNING: <details>`

### implement_complete

1. Resolve state UUID for "Validation" state (from config `workflowStates.validation`)
2. Move ticket to "Validation"
3. Post comment: `🤖 **AI Implementation Log** — All phases completed. Implementation moving to AI validation. Summary: [summary]`
4. Return: `SUCCESS` or `WARNING: <details>`
