You are tasked with installing a new skill from a GitHub repository into this project's skill system. This command fetches a specific skill folder from a remote repo and integrates it into the build pipeline.

2. **degit shorthand**: `owner/repo/skills/skill-name`
3. **degit shorthand with branch**: `owner/repo/skills/skill-name#branch`

## Process Steps

### Step 1: Parse the Input

Take the user-provided URL or shorthand and convert it to a valid `degit` path.

**URL Parsing Rules:**

1. If input starts with `https://github.com/`:
   - Strip the `https://github.com/` prefix
   - Look for `/tree/<branch>/` pattern (e.g., `/tree/main/`, `/tree/master/`)
   - If found: remove `/tree/<branch>/` and keep `owner/repo/path/to/skill`
   - The branch can optionally be appended with `#branch` at the end if needed

2. If input is already in degit format (`owner/repo/path/to/skill`):
   - Use as-is

**Examples:**
| Input | Parsed degit path |
|---|---|
| `https://github.com/supabase/agent-skills/tree/main/skills/supabase-postgres-best-practices` | `supabase/agent-skills/skills/supabase-postgres-best-practices` |
| `https://github.com/schpet/linear-cli/tree/main/skills/linear-cli` | `schpet/linear-cli/skills/linear-cli` |
| `supabase/agent-skills/skills/supabase-postgres-best-practices` | `supabase/agent-skills/skills/supabase-postgres-best-practices` |
| `owner/repo/skills/my-skill#develop` | `owner/repo/skills/my-skill#develop` |

**Derive the skill folder name** from the last path segment (e.g., `supabase-postgres-best-practices` from `.../skills/supabase-postgres-best-practices`).

Ask the user to confirm or override the skill folder name:

## Error Handling

- **degit fails**: Check if the path is correct, suggest removing `/tree/<branch>/` if user provided a GitHub URL
- **Empty folder**: Warn the user, suggest checking repo structure
- **Missing SKILL.md**: Ask user whether to keep or remove
- **Manifest write fails**: Report the error, suggest manual update
- **Build fails**: Report the error, the skill files are in place but not built

## Important Notes

- Always strip `/tree/<branch>/` from GitHub URLs before passing to degit
- The skill folder name should be concise and descriptive
- Never download entire repositories — only the specific skill folder
- The `Skills/` folder is the source of truth; the build copies to `.opencode/skills/`
- If the skill has subfolders (references, rules, scripts), they will be included automatically by the build
